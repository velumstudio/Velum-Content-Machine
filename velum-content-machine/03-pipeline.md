# 03 — Pipeline
> The weekly operating loop. One session of generation, one session of production, one review.
> Total weekly overhead target: ≤ 3 hours. If it takes more, the system is too heavy — cut, don't optimise.

---

## Weekly rhythm

**Monday — Generation session (Claude, ~60 min)**
1. Pick the week's input: one real signal (discovery call, DM, Emplifi observation) — see topic sourcing in 01
2. Run the agents for this week's 3 slots (or repurposing agent if a validated piece exists)
3. Approve/iterate by elimination — everything approved before leaving the session
4. Log approved briefs in Notion content pipeline (status: `ready`)

**Tuesday/Wednesday — Production session (~60–90 min)**
1. Carousels/statements → Canva, per `02-format-specs.md` (photos from the Midjourney working-moments library — generate new only when the library lacks the subject)
2. Approved Vera script → Higgsfield, locked formula, single generation attempt budgeted
3. Export, final visual check against v5 (typography, gold ≤8%, watermark on every slide)
4. Notion status → `scheduled`, dates assigned

**Publishing — native Instagram scheduling.** No third-party scheduler until 3 Blueprint clients (locked in README).

**Friday — Performance review (Emplifi, ~20 min)**

| Metric | Why it matters |
|---|---|
| Saves | ICP recognition — the primary quality signal |
| DMs containing 'BLUEPRINT' | The only conversion metric that counts |
| Profile visits → follows | Funnel top working |
| Reach into non-followers | Distribution health, secondary |

**Decision rules:**
- A piece with outsized saves or a 'BLUEPRINT' DM → mark `validated` in Notion → feeds repurposing agent
- Two consecutive weeks of a pillar underperforming → change the *angle*, never the cadence
- Vanity spikes (likes without saves/DMs) → note and ignore

## Review gates (nothing skips these)

1. **Approval gate** — no production before Paulina approves the brief/script in Claude. (This directly counters the late-follow-up pattern: approval and production are scheduled sessions, not open loops.)
2. **Brand gate** — pre-export check: v5 typography, banned patterns absent, footer + watermark present, one CTA
3. **Vera gate** — script carries `Status: APPROVED` before any generation credit is spent

## Escalation triggers (when to change the system — not before)

| Trigger | Then |
|---|---|
| 3 validated Blueprint clients | Unlock: scheduler evaluation, cadence increase, ICP V1 review |
| 4+ weeks of consistent 3/week publishing | Unlock: code layer for agents (parse output contracts → Notion via MCP) |
| A pillar produces 2 'BLUEPRINT' DMs | Double its slot share for 2 weeks, then reassess |
| Weekly overhead exceeds 3 hours for 2 weeks | Cut a format. Do not add tooling to compensate. |

## What this pipeline refuses to do

No batch production beyond 2 weeks. No content calendar theater. No new
platforms. No A/B infrastructure. The machine's output is DMs that say
'BLUEPRINT' — everything else is instrumentation.
