# 03 — Pipeline
> The weekly operating loop. One input. Three posts. One approval gate.

---

## The loop — one cycle

```
MONDAY
  Input: one topic (client signal / framework component / operator observation)
  Action: run the relevant agent in Claude → structured brief or script
  Gate: Paulina reviews output → approves, cuts, or requests one revision

TUESDAY–WEDNESDAY
  Production: approved brief → Canva (carousel/statement) or Higgsfield (Vera)
  Gate: final asset reviewed before scheduling

THURSDAY–SATURDAY
  Publish: 3 posts across the week per the slot system below
  Measure: note saves, DMs, ICP profile visits (not vanity metrics)

SUNDAY
  Review: 15 minutes — what performed? what generated a DM or a follow?
  Input for next cycle: one observation from the week becomes next Monday's topic
```

## Weekly slot system

| Slot | Day | Format | Pillar |
|---|---|---|---|
| A | Monday or Tuesday | Carousel or statement post | Authority |
| B | Wednesday or Thursday | Carousel or reel | Education |
| C | Friday or Saturday | Rotates weekly | Offer (weeks 1, 3) · Process (weeks 2, 4) |

**Vera videos:** slot into B or C when a script is approved. Max 1 Vera video per week. Never force a Vera video — only when the topic fits the locked formula.

**Stories:** opportunistic. Zero production overhead. No planning required. Do not count toward the 3-post minimum.

## Agent selection — which agent to run

| Input type | Agent |
|---|---|
| New topic → carousel | `carousel-agent.md` |
| New topic → reel (faceless) | `reel-agent.md` |
| New topic → Vera video | `vera-script-agent.md` |
| Validated content → more distribution | `repurposing-agent.md` |

**Repurposing agent runs only on pieces with a measurable performance signal.** Minimum 7 days after publication. Do not repurpose on assumption.

## Review gate — what Paulina approves

Every piece passes through exactly one approval gate before production:

1. **Brief/script approval** — agent output reviewed in Claude. Paulina approves or requests one revision. If the revision doesn't resolve the issue, start with a new input rather than iterating further.
2. **Asset approval** — final Canva or Higgsfield output reviewed before publish. Not optional.

**Nothing goes to Canva or Higgsfield without brief approval. Nothing publishes without asset approval.**

## Publish checklist (run before every post)

- [ ] `@thevelum.studio` watermark present on every slide/frame
- [ ] { • } mark on cover and closing slides
- [ ] One CTA only — from the approved library — on the closing slide/beat
- [ ] No banned vocabulary, no banned patterns
- [ ] Caption: hook extension + argument + CTA + footer standard
- [ ] Max 5 hashtags, below the footer
- [ ] Vera videos: AI disclosure confirmed on-camera or in caption
- [ ] File named and saved to the Velum Notion content inbox

## Performance review (Sunday, 15 minutes)

What to track (via Emplifi weekly review):
- Saves — best signal for Education and Authority content
- DMs mentioning 'BLUEPRINT' or content-specific language
- ICP-profile follows (check manually — follower quality not quantity)
- Carousel slide-through rate — which slide loses people

What not to track:
- Likes (not a conversion signal)
- Follower count week-to-week (too noisy at this stage)
- Reach on individual posts (too early to optimise for distribution)

**One observation per review → next week's input.** If nothing performed, the observation is "what didn't land and why" — still a valid input.

## Hard locks (do not remove until explicitly unlocked)

These constraints exist until **3 validated Blueprint clients** are confirmed:

| What | Why |
|---|---|
| No scheduling automation (Metricool, Buffer, Later) | Manual publishing keeps the loop human and observable at this stage |
| No new content formats | Carousel + reel + Vera = the validated stack. Novelty before volume is a trap. |
| No platforms beyond Instagram | Distribution before depth is a trap. One platform, done well. |
| No batch production beyond 2 weeks ahead | Planning horizon beyond 2 weeks optimises for a future state that doesn't exist yet |
| No repurposing without performance signal | Multiplying noise is not a strategy |

**If a session starts drifting toward building instead of publishing — this file is the pushback. Publish the next post.**

## When things break

**Topic blocked:** Run the repurposing agent on the best-performing piece from the last 30 days. One real signal → one derivative → 2 posts covered.

**Production blocked (Canva time, Higgsfield credits):** Publish a statement post. It requires only a brief, a phone screenshot of the Notion card, and the caption. Do not skip a week because production was hard.

**Approval loop stalling:** If a brief requires more than one revision, abandon it and pick a new input. The right topic briefs cleanly. A topic that resists structure is not the right topic yet.
