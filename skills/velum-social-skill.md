## Velum's Instagram system

### The one job

Every piece of content must trace to one of three outcomes:
1. Make the ICP recognise her commercial gap (Authority / Education)
2. Make Velum the obvious structural answer (Process / Offer)
3. Convert recognition into a DM or a booked call (Offer / Social Proof)

If a piece doesn't map to one of these — don't make it.

### The five pillars

| Pillar | Job | Primary formats |
|---|---|---|
| Authority | Name the problem precisely. "This person has seen this before." | Carousel, statement post |
| Education | Teach the framework. "I didn't know this had a name." | Carousel, reel |
| Process | Studio in action. "They actually do this." | Reel (B-roll), story |
| Offer | Direct. Specific. Reader knows exactly what to do. | Carousel, reel |
| Social Proof | Results over adjectives. Anonymised until explicit permission. | Reel, carousel |

### Format stack

| Format | Use when | Agent |
|---|---|---|
| Carousel (4–7 slides) | Multi-claim argument, education, offer breakdown | carousel-agent.md |
| Statement post (single image) | One sharp claim that stands alone | No agent — brief inline |
| Reel (faceless, 15–30s) | Process, hooks, fast concepts | reel-agent.md |
| Vera video (30–45s) | Spokesperson authority, diagnosis delivery | vera-script-agent.md |

### Hook doctrine

A hook is the ICP's exact situation, stated as fact, with no cushioning.

**Patterns that work:**
- The reframe: "X isn't the problem. X-architecture is."
- The distinction: "One service is a gig. A ladder is a business."
- The uncomfortable census: "Posting for months. Growing audience. Still no offer."
- The precise number: "40K followers. €0 in structured revenue."

**Never open with:** a question that flatters ("Have you ever wondered…"), a motivational observation ("So many creators are…"), or a softened version of the real problem.

### Caption structure (all feed formats)

```
Line 1 — extends the hook (never repeats it verbatim)
Body  — 2–4 sentences, one argument, one concrete outcome named
CTA   — one, from the approved library
— Velum — @thevelum.studio —
[Hashtags: max 5, below footer, niche-precise]
```

### CTA library

- `DM 'BLUEPRINT' if this is where you are.`
- `Save this. Come back when you're ready to build.`
- `Follow for the operator point of view.`
- `Tag a creator who needs to hear this.`
- `3 spots this cycle. Book below.` *(only when literally true)*

## Content planning workflow

### For a single piece

1. Identify: topic → pillar → format
2. Check: does it serve one of the three outcomes above?
3. Run the relevant agent → structured brief
4. Review against the six principles (below)
5. Return brief — nothing goes to production without approval

### For a weekly batch

Default batch: 3 posts per week

- Slot A (Authority): carousel or statement post
- Slot B (Education): carousel or reel
- Slot C (rotates): Offer (weeks 1, 3) · Process (weeks 2, 4)

Vera slots into B or C when a script is approved. Max 1 Vera per week. Never force it.

## The six content principles — pre-publish gate

Every piece must pass all six. One failure = rewrite, not proceed.

1. **Name a tension the ICP already feels.** Recognition in the first three seconds.
2. **Communicate without convincing.** Velum states; the right client self-selects.
3. **Outcomes over vibes.** Name a concrete change — offer architecture, monetisation plan, 30-day sequence.
4. **Earn the scroll in the first line.** The hook is the problem stated precisely.
5. **One CTA. Named. Specific.** Never stack.
6. **Faceless always.**
